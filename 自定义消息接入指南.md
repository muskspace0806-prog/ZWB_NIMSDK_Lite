# 自定义消息 Cell 接入指南

本项目基于 **NIMSDK_LITE**，会话列表和单聊页面均为纯自定义 UI。  
本文档说明如何新增一种自定义消息类型并渲染为独立的 Cell。

---

## 架构概览

```
服务端下发自定义消息（JSON）
        ↓
ZWB_CustomAttachmentParser.parse()   // 解析 JSON → ZWB_CustomAttachment
        ↓
ZWB_CustomAttachment                 // 统一附件模型，含 cellType + 业务字段
        ↓
ZWB_ChatViewController.cellForRowAt  // 根据 cellType 分发到对应 Cell
        ↓
ZWB_ImageTextCell / ZWB_XxxCell      // 各自渲染自己的 UI
```

---

## 服务端消息 JSON 格式

```json
{
  "first":  10,
  "second": 101,
  "data": {
    "title":  "标题",
    "desc":   "描述文字",
    "picUrl": "https://example.com/image.jpg",
    "webUrl": "https://example.com"
  }
}
```

| 字段 | 说明 |
|------|------|
| `first` | 一级类型，与 `second` 组合唯一标识消息类型 |
| `second` | 二级类型 |
| `data` | 业务字段，Cell 按需取用 |

## 新增自定义消息类型（4 步）

以新增一个**红包消息**（first=30, second=1）为例：

### 第 1 步：在 `ZWB_CellType` 加 case

文件：`ZWB_NIMSDK_Lite/自定义Cell/ZWB_CustomMessage.swift`

```swift
enum ZWB_CellType {
    case imageText   // 已有
    case redPacket   // ← 新增：红包消息
}
```

### 第 2 步：在 `from()` 加映射

同文件：

```swift
static func from(first: Int, second: Int) -> ZWB_CellType? {
    switch (first, second) {
    case (10, 101): return .imageText
    case (30, 1):   return .redPacket   // ← 新增
    default:        return nil
    }
}
```

### 第 3 步：新建 Cell 文件

在 `ZWB_NIMSDK_Lite/自定义会话 + 单聊/Cells/` 目录下新建 `ZWB_RedPacketCell.swift`：

```swift
import UIKit
import SnapKit

class ZWB_RedPacketCell: ZWB_BaseChatCell {

    static let reuseId = "ZWB_RedPacketCell"

    // 1. 声明 UI 控件
    private let amountLabel: UILabel = {
        let lb = UILabel()
        lb.font = .systemFont(ofSize: 18, weight: .bold)
        lb.textColor = .white
        return lb
    }()

    // 2. init 中把控件加到 bubbleView
    override init(style: UITableViewCell.CellStyle, reuseIdentifier: String?) {
        super.init(style: style, reuseIdentifier: reuseIdentifier)
        bubbleView.backgroundColor = .systemRed
        bubbleView.addSubview(amountLabel)
        amountLabel.snp.makeConstraints {
            $0.edges.equalToSuperview().inset(UIEdgeInsets(top: 12, left: 16, bottom: 12, right: 16))
        }
    }

    required init?(coder: NSCoder) { fatalError() }

    // 3. 实现 configure，第一行调用 applyLayout
    func configure(attachment: ZWB_CustomAttachment, isSend: Bool) {
        applyLayout(isSend: isSend)
        amountLabel.text = attachment.desc ?? "红包"  // 取 desc 字段显示金额描述
    }
}
```

> **规则：**
> - 继承 `ZWB_BaseChatCell`
> - 所有控件 addSubview 到 `bubbleView`（不是 contentView）
> - `configure` 第一行必须调用 `applyLayout(isSend:)`

### 第 4 步：在 `ZWB_ChatViewController` 注册并分发

文件：`ZWB_NIMSDK_Lite/自定义会话 + 单聊/ZWB_ChatViewController.swift`

**4a. 注册 Cell（在 tableView 的 lazy 初始化里）：**

```swift
tv.register(ZWB_RedPacketCell.self, forCellReuseIdentifier: ZWB_RedPacketCell.reuseId)
```

**4b. 在 `cellForRowAt` 的 `.MESSAGE_TYPE_CUSTOM` 分支加 case：**

```swift
case .MESSAGE_TYPE_CUSTOM:
    let att = msg.attachment as? ZWB_CustomAttachment
    switch att?.cellType {
    case .imageText:
        let cell = tableView.dequeueReusableCell(...) as! ZWB_ImageTextCell
        cell.configure(attachment: att!, isSend: isSend)
        return cell
    case .redPacket:                                          // ← 新增
        let cell = tableView.dequeueReusableCell(
            withIdentifier: ZWB_RedPacketCell.reuseId,
            for: indexPath
        ) as! ZWB_RedPacketCell
        cell.configure(attachment: att!, isSend: isSend)
        return cell
    default:
        // 未知类型走兜底 Cell
        let cell = tableView.dequeueReusableCell(...) as! ZWB_DefaultMessageCell
        cell.configure(message: msg, isSend: isSend)
        return cell
    }
```

## 如果需要新增业务字段

`ZWB_CustomAttachment` 目前包含通用字段：`title`、`desc`、`picUrl`、`webUrl`。

如果新类型需要额外字段（如红包金额 `amount`），直接在 `ZWB_CustomAttachment` 里添加：

```swift
class ZWB_CustomAttachment: V2NIMMessageCustomAttachment {
    // ... 已有字段 ...
    var amount: String?   // ← 新增

    override func parse(_ attach: String) {
        // ... 已有解析逻辑 ...
        amount = inner["amount"] as? String   // ← 新增解析
    }
}
```

## 关键注意事项

| 事项 | 说明 |
|------|------|
| `ZWB_CustomAttachmentParser.register()` | 必须在 `setupIM` 中调用，否则收到的自定义消息无法解析 |
| `v2LocalConversationService` | 本地会话模式必须用此服务，不能用 `v2ConversationService` |
| `onSyncFinished` | 会话列表必须等此回调后才能拉取，否则报 `misuse V2 API` |
| `option.onlyQueryLocal = false` | 拉取历史消息时必须设置，否则新账号本地无数据 |
| Cell 命名 | 不能用 `textLabel`、`imageView` 等 `UITableViewCell` 父类属性名 |
